# Sprint 5: Evaluation Dashboard and Final Presentation Polish

## Overview
Sprint 5 turns InfraSight into a dissertation-ready demonstration system. It adds a comprehensive evaluation dashboard, alert acknowledgement and resolution workflow, scenario validation history, settings, improved dashboard overview, documentation, and verification coverage.

## Evaluation Dashboard
The Evaluation page presents dissertation evidence using eight sections: baseline vs post-deployment KPI comparison, improvement summary cards, model performance metrics, confusion matrix, Random Forest feature importance, before/after bar chart, system activity summary, and JSON export.

## Alert Management
Predictive alerts are generated by the Random Forest pipeline when a device is classified as `fault_prone` above threshold. Sprint 5 adds alert filtering, acknowledgement, resolution, status updates, and management actions through backend endpoints and the frontend Alerts page.

## Scenario Validation
The scenario validation suite runs known failure modes against the simulator, builds telemetry windows, runs prediction, generates alerts, and records pass/fail run history. This demonstrates that model outcomes align with expected scenario classes.

## Dissertation Evidence
| Feature | Research Objective Evidence |
|---|---|
| KPI comparison | Shows operational improvement after proactive monitoring |
| Model metrics | Demonstrates classifier performance and validity |
| Confusion matrix | Shows correctness by actual/predicted class |
| Feature importance | Provides explainability for Random Forest decisions |
| Scenario validation | Proves expected fault modes are detected |
| Alert workflow | Demonstrates actionable incident response |
| System activity summary | Shows live system adoption and monitoring volume |

## How to Demo
1. Login with the configured Firebase admin account.
2. Open Dashboard for the GZU infrastructure overview.
3. Review device health cards, alerts, predictions, and system health chart.
4. Open a device detail page and run telemetry simulation or Predict Now.
5. Open Predictions to show model metrics, feature importance, and confusion matrix.
6. Open Alerts to acknowledge and resolve predictive alerts.
7. Open Scenarios, run a scenario, and show pass/fail validation plus history.
8. Open Evaluation and walk through KPI improvements and export the report.
9. Open Settings to show thresholds, model settings, and Firebase status.
